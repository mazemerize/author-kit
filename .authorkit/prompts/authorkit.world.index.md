## Goal

Build or rebuild `world/_index.md` — a central lookup index generated by PowerShell with zero LLM cost. The index enables fast entity lookups, alias resolution, and targeted world/ file loading across all commands.

## Outline

1. **Setup**: Run `{{SCRIPT_BUILD_WORLD_INDEX}}` from repo root and parse BOOK_DIR. All paths must be absolute.

2. **Determine mode from user input**:
   - **(no input or "rebuild")**: Run `build-world-index.ps1 -Json` to perform a full rebuild.
   - **"add-frontmatter"**: Run `build-world-index.ps1 -AddFrontmatter -Json` to add YAML frontmatter to files that lack it, then rebuild the index.

3. **Run the PowerShell script** from the repo root:
   ```
   .authorkit/scripts/powershell/build-world-index.ps1 -Json
   ```
   Or with `-AddFrontmatter` if requested.

4. **Parse JSON output** and report results:
   - INDEX_FILE: path to the generated `_index.md`
   - ENTITY_COUNT: total entities indexed
   - ALIAS_COUNT: total alias entries
   - CHAPTER_COUNT: chapters tracked in the manifest
   - FILES_WITHOUT_FRONTMATTER: count of files missing structured metadata

5. **Report completion**:
   - Summary of what was generated
   - If FILES_WITHOUT_FRONTMATTER > 0, suggest running with `add-frontmatter` to upgrade legacy files
   - Suggest next step: `/authorkit.world.verify` to check consistency, or continue with chapter work

## Key Rules

- **This command never modifies world/ entity files** (except in `add-frontmatter` mode, where it prepends YAML frontmatter to files that lack it).
- **The index is a cache, not a source of truth.** If `_index.md` is deleted, it can be rebuilt from scratch with zero data loss.
- **The index should be rebuilt** after any command that modifies world/ files: `world.build`, `world.update`, `retcon`, `pivot`, `chapter.reorder`.
- See `.authorkit/templates/world-entity-frontmatter.md` for the frontmatter schema reference.
